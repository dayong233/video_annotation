# 标注规则


# 🎥 精细化人体动作与物理原理标注规范

本项目旨在构建一个高精度的多模态数据集，重点关注**微观动作**与动作背后的 **物理力学机制** 。

## 1. 数据结构 (JSON Schema)

标注文件以视频为单位，核心字段定义如下：

**JSON**

```
{
  "video_name": "视频文件名.mp4",
  "action": [
    {
      "start_time": 0.0,  // 精确到小数点后两位，实际标注中按帧拆分后转为秒
      "end_time": 0.8,
      "description": "客观动作描述：解剖学细节 + 接触几何形态",
      "reason": "物理/逻辑解释：力学原理 + 任务意图"
    }
  ]
}
```

---

## 2. 📝 Description 标注规范 (动作描述)

**核心原则：客观、解剖学精确、动态细节。**

* **主语统一：** 操作者统称为  **"记录者"** 。
* **左右区分：** 必须明确指出是  **"左手/右手"** 、 **"左/右侧"** 。
* **手部精细度：** 拒绝笼统的"拿"或"取"。必须描述手指状态：
  * ✅ *“拇指按在...，四指内扣...”*
  * ✅ *“食指与拇指捏住...”*
  * ✅ *“手掌向下，手腕内旋...”*
* **接触几何：** 描述手与物体的接触点（如： *杯柄空洞、把手内侧、瓶盖边缘* ）。

**推荐句式：**

> `[身体部位]` + `[手势形态]` + `[接触/施力动词]` + `[物体具体部位]`。

---

## 3. 🧠 Reason 标注规范 (原因解释)

**核心原则：物理优先 (Physics First)，逻辑为辅。**

解释“为什么要这样操作”时，请优先从力学角度分析，其次是任务逻辑。

### 3.1 物理力学维度 (High Priority)

* **摩擦力 (Friction):** 解释抓握方式（e.g.,  *“增加接触面积以防滑落”* ）。
* **力矩与杠杆 (Torque/Leverage):** 解释开门、拧盖、按压手柄（e.g.,  *“利用杠杆原理开启阀门”、“克服铰链阻尼”* ）。
* **重力 (Gravity):** 解释倒水、垂手（e.g.,  *“利用重力排空液体”、“利用势能转化为动能”* ）。
* **螺旋/机械结构:** 解释旋转方向（e.g.,  *“遵循螺旋副锁紧方向”、“挤压密封圈”* ）。

### 3.2 任务逻辑维度

* **预备:**  *“为下一步倒牛奶腾出空间”* 。
* **安全:**  *“激活微波炉安全互锁机制”* 。

## 4. ✅ 标注示例 (Good vs Bad)

### 场景：打开冰箱拿牛奶

| **维度**        | **❌ 错误示例 (Too Simple)** | **✅ 正确示例 (Detailed & Physics-based)**                                                                          |
| --------------------- | ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| **Description** | 他打开冰箱，拿出牛奶。             | 记录者**伸出左手** ，**拇指与四指分开**握住冰箱把手向外拉开。紧接着**右手四指并拢内扣**牛奶桶把手内侧。 |
| **Reason**      | 因为要喝牛奶。                     | **抓取把手：把手是重心的最佳施力点** ，确保移出重物时的稳定性；**向外拉动：**克服磁性密封吸力。                     |

### 场景：倒掉杯子里的水

| **维度**        | **❌ 错误示例 (Too Simple)** | **✅ 正确示例 (Detailed & Physics-based)**                   |
| --------------------- | ---------------------------------- | ------------------------------------------------------------------ |
| **Description** | 把水倒进水槽。                     | 左手**手腕顺时针内旋约120°** ，将马克杯倒置，使水流出。     |
| **Reason**      | 清空杯子。                         | **手腕内旋：利用重力**排空杯内液体，防止残留生水与饮品混合。 |

## 5. 🔑 常用术语速查表

在撰写 `Reason` 时，请尽量高频使用以下词汇：

* **力学:** 摩擦力、力矩、杠杆原理、重力线、惯性、阻尼、反作用力、施力点。
* **动作:** 内旋/外旋、顺时针/逆时针、压缩、形变、对齐、悬空。
* **目的:** 最小化体力、最大化接触面、物理密封、安全互锁。

### 6.2 建议检查与优化方向
如果您选择参考预标注数据，请重点关注以下维度并按需调整：

1.✂️ 时间轴确认 (Segmentation Check):

拆分与合并： 模型可能会将复杂的连续动作（如“走向冰箱”+“拉开门”）合并为一个长片段，或者将连贯动作切得太碎。请依据动作的语义完整性调整时间切片。

2.👁️ 事实性修正 (Fact Verification):

左右校准： 模型常在左右手/左右脚的判断上出现幻觉，必须依据视频画面实际情况进行修正。

细节补全： 检查是否遗漏了关键的手指姿态（如“内扣”、“捏住”）或接触点细节。

3.🔧 描述与逻辑精修 (Refinement):

物理化润色： 预标注的 reason 可能缺乏具体的物理力学术语（如摩擦力、力矩、重力）。请参照第 3 点规范进行补充。

去伪存真： 确保解释内容与视频画面严格对应，剔除看似专业但实际未发生的过度解读。
